\section{Executive Summary}

The EDOS Sales Dashboard is a comprehensive full-stack web application designed for e-commerce analytics and behavioral intelligence. This platform provides real-time insights into sales performance, order management, cancellation analytics, and advanced behavioral tracking with predictive capabilities.

\subsection{Key Highlights}
\begin{itemize}[leftmargin=*]
    \item \textbf{Technology Stack:} React 19, Node.js/Express, SQLite, deployed on Render
    \item \textbf{Core Features:} Real-time analytics, behavioral simulation, purchase prediction, smart recommendations
    \item \textbf{Deployment:} Fully automated CI/CD pipeline with GitHub integration
    \item \textbf{Authentication:} Role-based access control (Admin/User)
    \item \textbf{Database:} SQLite with automated migrations and schema versioning
\end{itemize}

\subsection{Project Objectives}

\begin{enumerate}[leftmargin=*]
    \item Provide comprehensive sales analytics and KPI tracking
    \item Enable order management with CRUD operations
    \item Analyze cancellation patterns and revenue impact
    \item Implement behavioral tracking and purchase prediction
    \item Deliver actionable insights through data visualization
    \item Create a scalable, production-ready platform
\end{enumerate}

\section{Database Architecture}


\subsection{Database Schema}

The application uses SQLite with the following core tables:

\subsubsection{Users Table}
\begin{lstlisting}[language=SQL]
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    role TEXT DEFAULT 'user',
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
\end{lstlisting}

\subsubsection{Orders Table}
\begin{lstlisting}[language=SQL]
CREATE TABLE orders (
    order_id TEXT PRIMARY KEY,
    customer_name TEXT NOT NULL,
    product_name TEXT NOT NULL,
    category TEXT NOT NULL,
    order_value REAL NOT NULL,
    order_date TEXT NOT NULL,
    payment_method TEXT,
    region TEXT,
    status TEXT DEFAULT 'Pending',
    delivery_time INTEGER,
    cancellation_reason TEXT,
    created_by TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
\end{lstlisting}

\subsubsection{Behavioral Logs Table}
\begin{lstlisting}[language=SQL]
CREATE TABLE behavioral_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    action TEXT NOT NULL,
    hesitation_score REAL DEFAULT 0,
    details TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
\end{lstlisting}

\subsubsection{Schema Version Table}
\begin{lstlisting}[language=SQL]
CREATE TABLE schema_version (
    version INTEGER PRIMARY KEY,
    applied_at TEXT DEFAULT CURRENT_TIMESTAMP
);
\end{lstlisting}

\subsection{Database Migrations}

Automated migration system ensures schema consistency:

\begin{itemize}[leftmargin=*]
    \item \textbf{Version Tracking:} Schema version table tracks applied migrations
    \item \textbf{Idempotent Migrations:} Safe to run multiple times
    \item \textbf{Production Safety:} Migrations run automatically on deployment
    \item \textbf{Column Additions:} ALTER TABLE statements for schema updates
\end{itemize}

\textbf{Example Migration (Cancellation Reason):}
\begin{lstlisting}[language=SQL]
-- Check if migration already applied
SELECT version FROM schema_version WHERE version = 1;

-- Add cancellation_reason column if not exists
ALTER TABLE orders ADD COLUMN cancellation_reason TEXT;

-- Record migration
INSERT INTO schema_version (version) VALUES (1);
\end{lstlisting}

\section{Authentication \& Security}


\subsection{Authentication Flow}

\begin{enumerate}[leftmargin=*]
    \item User submits credentials (email/password)
    \item Server validates credentials against database
    \item Password verified using bcrypt comparison
    \item JWT token generated with user payload
    \item Token returned to client and stored in localStorage
    \item Subsequent requests include token in Authorization header
    \item Server validates token on protected routes
\end{enumerate}

\subsection{Password Security}

\begin{itemize}[leftmargin=*]
    \item \textbf{Hashing Algorithm:} bcryptjs with salt rounds
    \item \textbf{Storage:} Only hashed passwords stored in database
    \item \textbf{Validation:} Secure comparison using bcrypt.compare()
    \item \textbf{No Plain Text:} Passwords never stored or transmitted in plain text
\end{itemize}

\subsection{JWT Token Management}

\begin{itemize}[leftmargin=*]
    \item \textbf{Token Generation:} jsonwebtoken library
    \item \textbf{Payload:} User ID, email, role
    \item \textbf{Expiration:} Configurable token lifetime
    \item \textbf{Secret Key:} Environment variable (JWT\_SECRET)
    \item \textbf{Verification:} Middleware validates token on each request
\end{itemize}

\subsection{Role-Based Access Control (RBAC)}

\begin{itemize}[leftmargin=*]
    \item \textbf{Roles:} Admin and User
    \item \textbf{Admin Privileges:}
    \begin{itemize}
        \item Full CRUD operations on orders
        \item User management
        \item Access to simulator and diagnostics
        \item System-wide configuration
    \end{itemize}
    \item \textbf{User Privileges:}
    \begin{itemize}
        \item View dashboards and analytics
        \item View orders (read-only)
        \item Access personal settings
    \end{itemize}
\end{itemize}

\subsection{Security Best Practices}

\begin{itemize}[leftmargin=*]
    \item CORS configuration for cross-origin requests
    \item Environment variables for sensitive data
    \item Input validation and sanitization
    \item SQL injection prevention (parameterized queries)
    \item XSS protection through React's built-in escaping
    \item HTTPS enforcement in production
\end{itemize}

\section{Challenges \& Solutions}


\subsection{Challenge 1: Missing Database Column in Production}

\textbf{Problem:} The cancellation\_reason column existed in schema definition but not in production database, causing API failures.

\textbf{Root Cause:} Database created before column was added to schema.

\textbf{Solution:}
\begin{itemize}[leftmargin=*]
    \item Implemented automated migration system
    \item Added schema\_version tracking table
    \item Created idempotent ALTER TABLE migration
    \item Deployed migration with zero downtime
\end{itemize}

\textbf{Outcome:} Production database updated successfully, API working correctly.

\subsection{Challenge 2: Chart Rendering Issues}

\textbf{Problem:} Recharts library not rendering in production build (React 19 compatibility).

\textbf{Attempted Solutions:}
\begin{itemize}[leftmargin=*]
    \item Updated Recharts to latest version
    \item Tried different chart types
    \item Investigated React 19 compatibility
\end{itemize}

\textbf{Final Solution:}
\begin{itemize}[leftmargin=*]
    \item Implemented custom CSS-based horizontal bar charts
    \item Used CSS animations for smooth transitions
    \item Created responsive design with flexbox
    \item Added color-coded visual indicators
\end{itemize}

\textbf{Outcome:} Charts now render reliably with better performance and customization.

\subsection{Challenge 3: Deployment Cache Issues}

\textbf{Problem:} Frontend changes not appearing after deployment.

\textbf{Root Causes:}
\begin{itemize}[leftmargin=*]
    \item Browser cache holding old assets
    \item Render build cache
    \item Service worker caching
\end{itemize}

\textbf{Solutions:}
\begin{itemize}[leftmargin=*]
    \item Hard refresh (Ctrl+F5) for users
    \item Incognito mode for testing
    \item Manual deployment trigger on Render
    \item Cache-busting with Vite build hashes
\end{itemize}

\subsection{Challenge 4: API Context Export}

\textbf{Problem:} CancelInsights page couldn't access API methods.

\textbf{Root Cause:} AuthContext wasn't exporting the api object.

\textbf{Solution:}
\begin{itemize}[leftmargin=*]
    \item Added api to AuthContext provider value
    \item Exported useAuth hook with api access
    \item Updated components to use api from context
\end{itemize}

\textbf{Outcome:} All pages can now access API methods consistently.
